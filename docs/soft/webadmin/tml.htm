 2.平台功能简介和协议说明
   本软件用于监控由Microsoft IIS或Apache Web Server创建的Web服务器。通过实时监控服务的运行状态,一旦Web服务发生错误（程序500错误或被DDOS死掉），能自动重新启动IIS或Apache、或是通过请求指定的页面、或者运行指定程序、或者直接发邮件通知管理员。也可以重起服务器后然后自动登陆服务器并自动开启服务。本程序编写过程中，尽量占用最小资源，所以不用担心影响到服务器的性能。
本软件可任意传播和复制，但请保留本软件完整性，不允许修改或反汇编。(功能简介)
   本软件最早版本是我们在管理NT4的ASP网站时编写的一个很小 的监控程序，后来偶尔传到网络上，不断有热心的朋友给我们提出改进的意见，经过数次功能的完善而得到了现在这个软件。很高兴得知目前国内有数千家网站使用本程序来管理，其中还有很多著名网站，冰冻三尺非一日之寒，感谢大家三年多来一直使用这个的软件，毕竟继续的完善也要大家继续的建议和支持。(协议说明)

 4.购买方法
1，如何实现监控服务器
本程序实际上是通过本身定时访问指定服务器上的网页程序，然后根据服务器返回信息来判断服务器状态的。例如返回200，表示服务器正常，500表示服务器内部错误等。当Web服务器出现内部错误或程序死掉或被人黑掉等情况后，可以按您事先的安排采取下面的措施：
①自动重起Web服务,事实上大部分服务器错误都可通过重启WEB服务来恢复。②重起/关闭计算机,解决服务进程管理也失效的情况 。
③还原页面,如果你首页经常被黑,那么你监控这个页面,当被黑的时候,程序自动还原,达到一定防黑效果。如果你想保护整个网站不被黑，请用我们的另外一套“ＡｕｔｏＢａｃｋｕｐ网络数据备份系统”彻底保护整个网站。
④访问指定页面,当出错时,访问另一个你自己编写的有扩展功能的页面。
⑤运行本地程序,当服务器出问题时,自动运行本地某个程序,这个程序可以是你自己编写的,建议配合我们的其他六套软件使用,效果最好.
⑥发邮件给管理员,如果您监控的是远程Web服务器,或同时监控多台Web服务器,当然无法实现自动重启是不可能的,但是您这个把这个服务器的错误报告发email给管理员,来提醒管理员来处理,还支持群发和匿名发信也许以后会加个短信提醒,服务器错误后,自动发短信给管理员
2，如何设置参数
“设置程序所在位置”相当于一个安装程序，帮您安装到指定目录，并创建快捷方式。“系统启动后自动登录服务器”可以使服务器不显示登陆框，直接进入桌面，这样就可以运行很多前台服务，毕竟服务器很多重要的软件还需要登陆后才可以运行，比如“金山毒霸2000”和“天网防火墙”。其他的设置都是一看就明白的了。
3，如何设置监控任务
这里的设置也都是一看就明白的，还有一些技巧，比如一台服务器可能安装IIS和Apache，这样只需分别添加两个任务就可同时监控它们。另外“访问指定页面”和“运行本地程序”能够扩展功能，您可以自己制作一个具有特殊功能的页面或程序放到这里，当服务器错误时，让本程序帮您运行。还有一个技巧是监控远程服务器，您可以将远程服务器的web页面地址添加到任务，当远程服务器死掉后，让本程序发email给你。可以同时监控多台远程服务器，只要添加多个任务就可以了。发邮件的内容就是“邮件设置”里，可以发匿名信或群发，因此您也可以用本软件群发邮件。
4，什么是监控模板
监控模板可以帮您编写指定类型的网页监控程序，您也可以用服务器上现有的页面，比如用论坛首页，江湖首页来监控论坛或江湖是否死掉。
5，为什么无法运行程序
本程序运行时自动检查是否已经运行，如果运行了，就不会再次运行了。本程序窗体名称是NWA，意思是NYS Web Admin，如果系统中存在名称为NWA的窗体，本程序就不会运行了，有碰巧的时候会遇到，所以1.04版去掉了该功能。
6，几点补充
首先，监控URL请填写正确，以免服务器无故重起。其次，监控频率精确到5分钟，所以建议填写5的倍数，不够会自动进位，例如18分钟其实是20分钟检测一次。最后，本软件全部数据储存在注册表的\HKEY_LOCAL_MACHIN\Software\TML\TMLwebadmin\，此外不修改任何系统 文件，必要时可自行修改。
