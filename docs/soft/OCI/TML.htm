 2.功能简介
本套程序是为方便打包使用ORACLE开发的软件而制作的，能够完全独立连接Oracle，完全不需要SQL*Net，即装即连， 而现存的诸多驱动比如JDBC、ODBC、DOA（Direct Oracle Access）、ODAC（Oracle Data Access Components）都要依赖SQL*Net。
程序包含了我们二次开发并经过精简处理的ORACLE调用接口（Oracle Call Interface简称OCI） 主要提供给大家一组可对ORACLE数据库进行存取的接口函数。
并且我们利用OCI技术对应用系统工程环境ASEE
(Application System Engineering Environment)进行了功能扩充，
使其具有了访问ORACLE数据库的功能，接口简单高效，灵活易用。 本驱动是从事ORACLE开发的程序员首选组件，特别适合初学者，使用者已超过1500人，并被诸多公司购买用于ORACLE开发。 ORACLE调用接口(Oracle Call Interface简称OCI)提供了一组可对ORACLE数据库
进行存取的接口子例程(函数)，通过在第三代程序设计语言(如C语言)中进行调用
可达到存取ORACLE数据库的目的。 我们这套OCI开发方法实际上是将结构化查询语言(SQL)和第三代程序设计语言相结合的一种方法。
对数据库的访问是通过调用OCI库函数实现的，若将C语言作为宿主语言，
那么ORACLE库调用其实就是C程序中的函数调用，
一个含OCI调用的C程序其实就是用C语言编写的应用程序。
这样的程序既具有SQL语言非过程性的优点又具有C语言过程性的优点，
同时还可具有SQL语言的扩展――PL/SQL语言过程性和结构性的优点，
因此使得开发出的应用程序具有高度灵活性和强大的数据处理功能。 本安装包含我们使用Delphi6+ADO调用OCI编写的一个范例程序―一个Oracle管理器。
